## 1. 项目描述 (Project Description)

**FlowStudy** 是一款面向开发者的“AI 原生交互式学习平台”。 它打破了传统技术博客“作者单向输出、读者被动阅读”的死板模式，将深度技术文章（Java、C++、Go、数据库、算法等）与在线判题系统（OJ）无缝融合。其最大的独创性在于：系统能够在后台无感捕捉读者的学习时长、代码编写风格及报错轨迹，配合上下文感知的 AI 侧边栏助手，最终为每位用户自动提炼出极具个人色彩的专属学习笔记与能力画像。

## 2. 项目愿景 (Project Vision)

- **对于学习者（用户产品价值）**：打造一个绝对沉浸的“心流终端”。让学习者不需要在博客、IDE 和 ChatGPT 之间来回切屏。在这个平台里，代码可以即刻运行，疑问可以随时解答，而你只需专注思考，记录和总结的工作全部交给系统。
- **对于开发者（工程技术价值）**：构建一个具有极高技术壁垒的现代化系统。它不仅要解决传统后端的高并发处理、判题沙箱隔离与中间件异步解耦问题，还要真正落地大型语言模型（LLM）的工程化应用（Agent 编排与 RAG 检索），成为一个教科书级别的全栈/后端架构范例。

## 3. 核心功能模块 (Core Features)

系统由三个核心模块组成，形成一个完整的学习闭环：

### 1. 核心业务模块 (Core Service) —— 系统的“前厅大管家”

这是整个系统的基石，负责所有直接面向用户的常规业务和调度，需要极高的稳定性和并发处理能力。

- **推荐技术栈**：**Java (Spring Boot 3 + Spring MVC + MyBatis-Plus)**
- **核心功能清单**：
  1. **用户与权限体系**：处理用户的注册、登录，以及基于 JWT 或 Session 的安全鉴权。
  2. **内容分发引擎**：负责博客文章、章节结构、题库列表的增删改查与展示渲染。
  3. **高并发网关控制**：接收前端用户提交的代码请求。利用 **Redis + Lua 脚本**实现接口防刷限流，防止恶意提交压垮系统。
  4. **任务投递枢纽**：将用户的代码提交请求、页面停留时长等埋点数据，打包封装后安全地投递到 **RabbitMQ 消息队列**中，实现异步削峰。

### 2. 判题沙箱模块 (Judge Service) —— 系统的“后厨执行者”

这是一个纯粹的、不需要和前端用户打交道的后台服务，专门负责干最耗费 CPU、对安全隔离要求最高的“脏活累活”。

- **推荐技术栈**：**Go (调度层)** + **C/C++ (底层沙箱引擎，推荐直接调用开源实现)**
- **核心功能清单**：
  1. **队列消费**：作为一个安静的 Worker，持续监听 RabbitMQ，从队列中拉取等待判题的代码任务。
  2. **安全隔离运行 (Sandbox)**：将用户的代码扔进极其严格的 Linux 沙箱环境中编译和运行。通过底层机制限制其不能联网、不能读取本地敏感文件，并且精确限制代码的执行时间（防止死循环）和内存使用（防止内存溢出）。
  3. **结果比对与回传**：将代码的实际输出与标准测试用例进行比对，生成最终的运行状态（如 `Accepted` 通过, `Time Limit Exceeded` 超时, `Runtime Error` 报错），并将耗时、内存占用等结果回传给消息队列或数据库。

### 3. AI 分析与网关模块 (AI Agent Service) —— 系统的“最强大脑”

这是系统的灵魂所在，承载了“上下文感知”和“无感追踪生成”的最大亮点。

- **推荐技术栈**：**Python (FastAPI + LangGraph/LangChain)**
- **核心功能清单**：
  1. **侧边栏智能问答 (SSE 流式通信)**：提供与大型语言模型（如 Claude 或 GPT）对接的接口，处理前端侧边栏的对话请求。
  2. **上下文感知拦截**：在向大模型发送问题前，自动将用户当前的“阅读章节”和“最新报错代码”拼接到 Prompt（提示词）中，让 AI 能够精准作答。
  3. **用户行为数据清洗**：异步监听消息队列中用户的报错日志和提问历史，利用小模型或大模型的结构化输出能力，提炼出用户的“易错点”和“代码风格标签”。
  4. **个性化笔记生成引擎**：根据用户累积的数字画像和知识图谱，通过 Agent 工作流（Workflow），在章节结束时全自动生成带有极强个人色彩的 Markdown 学习笔记。



## 4. 核心创新点

**无感追踪与数字画像 (Invisible Tracking & Profiling) —— [核心卖点]**

- **异步采集**：无感记录页面停留时长、AI交互内容、代码提交频率、代码风格、易错点（如边界越界、空指针）。
- **画像更新**：后台 Agent 不断清洗这些行为数据，为用户打上动态能力标签。
- **自动生成笔记**：当章节结束或按周期触发时，结合用户的提问历史和高频错误代码，自动生成一份“带有你个人代码风格和思维习惯”的专属总结笔记。
